## Обоснование выбора метрик для оценки моделей

## **Для классификации:**

### **1. Accuracy (Точность)**

**Что измеряет:** Долю правильно классифицированных изображений среди всех изображений.

**Формула:** (TP + TN) / (TP + TN + FP + FN)

**Выбор обоснован:** Базовая метрика для общей оценки модели.

### **2. Precision (Точность)**

**Что измеряет:** Долю корректных предсказаний конкретного класса среди всех предсказаний этого класса.

**Формула:** TP / (TP + FP)

**Выбор обоснован:** Позволяет оценить, насколько точно модель определяет конкретный фрукт (например, сколько из предсказанных «яблок» действительно являются яблоками).

### **3. Recall (Полнота)**

**Что измеряет:** Долю правильно распознанных объектов конкретного класса среди всех реальных объектов этого класса.

**Формула:** TP / (TP + FN)

**Выбор обоснован:** Важна для оценки того, насколько хорошо модель находит все изображения определённого фрукта, минимизируя пропуски классов.

### **4. F1-Score (F-мера)**

**Что измеряет:** Гармоническое среднее между Precision и Recall

**Формула:** 2 × (Precision × Recall) / (Precision + Recall)

**Выбор обоснован:** Основная метрика для несбалансированных данных.
